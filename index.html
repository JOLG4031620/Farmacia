Lo básico sobre Sistemas Operativos y Redes de Trabajo

En está página web aprenderas los conceptos e información básica que debes saber sobre los distintos sistemas operativos y sus funciones, así como sus características, ventajas y desventajas, entre otras cosas.

Conceptos básicos

Hardware

Es la parte física de un ordenador o sistema informático. Está formado por los componentes eléctricos, eléctronicos y mecanicos.

Software

Representa toda la parte inmaterial o intangible que hace funcionar a un ordenador, para que realice una serie de tareas específicas.

Sistema Operativo

Gestiona y administra el hardware del dispositivo eléctronico, así como la ejecución de otros programas.

¿Qué es?

Se conoce como software comercial ó software propietario al tipo de software que tiene como fin una comercialización,es decir, una venta.


Características del software comercial:
  • No permite modificar el código fuente.
  • debes pagar por usarlo, generalmente a través de una licencia.
  • No se puede distribuir sin el permiso del propietario original que lo creó.
  • Ofrece buen nivel de experiencia de usuario y usabilidad.
  • Ofrece respaldo oficial de la empresa que lo desarrollo.

¿Qué es?

Es aquel software donde el usuario del programa tiene la libertad de copiarlo, modificarlo, redistribuirlo y distribuirlo para el beneficio de una comunidad.


Características del software libre:

  • Libertad de distribuir el programa, incluso copias de versiones modificadas.

Redes de Trabajo

Para conocer más sobre las redes de trabajo que existen debemos comprender algunos concetos clave.

Red

Es un sistema de comnicación que se da entre distintos equipos para poder realizar una comunicación eficiente, rapida y precisa, para la transmisión de datos de un ordenador a otro.

Red "Punto a Punto"

Las redes punto a punto son aquellas que responden a un tipo de arquitectura de red en las que cada canal de datos se usa para comunicar únicamente dos nodos, en contraposición a las redes multipunto, en las cuales cada canal de datos se puede usar para comunicarse con diversos modos.

Redes Lan

Estas redes están formadas por uno u varios segmentos de red conectados mediante dispositivos especiales. Son utilizadas por pequeñas y medianas empresas.

Redes WAN

Estas redes permiten unir redes a traves de varias ciudades, regiones, países. Son utilizadas por grandes empresas que tienen diseminada su organización a lo largo del mundo

Redes privadas

Una red privada es aquella en la que sus líneas de comunicación son diseñadas e instaladas por el usuario o administrador.

Redes públicas

las redes públicas son aquellas en las que las líneas de comunicación soportan información de diferentes usuarios.

Esta capa se encarga de mantener y controlar el enlace establecido entre dos computadores ue están transmitiendo datos de cualquier índole.
Para obtener éxito en el proceso de comunicación, la capa de sesión tiene que preocuparse con la sincronización entre hosts, para que la sesión abierta entre ellos se mantenga arriba.



Capa 4.- Transporte

La capa de transporte garantizada que los mensajes lleguen a su destinatario sin errores, en la secuencia correcta y sin perdidas de datos.
Los protocolos de las capas superiores no tienen cualquier preocupación por la transferencia de datos.
También esa capa se encarga de recibir los datos enviados or la capa de sesión.



Capa 3.- Red

La capa de red provee los medios funcionales y de procedimineto para que se haga la transferencia de tamaño variable de datos en secuencias de ub¿n origen en el host que se encuentra en una red de datos para un host que se encunetra en una red para un host de destino que se encuentra en una red de datos diferente, tratando de mantener la calidad de servicio e habría sido requerida por la capa de transporte.



Capa 2.- Enlace de datos

Proporciona tránsito de datos confiable a través de un enlace físico. Lo que permite que las capas superiores a ella, estén seguras de que la transmisión de datos a través del vínculo físico se va a realizar prácticamente sin errores.
Está capa se ocupa del direccionamiento físico del acceso al medio, de la detención de errores, de la distribución ordenada de tramas y del control del flujo.



Capa 1.- Física

La capa física define las específicaciones eléctricas, mecanicas, de procedimiento y funcionalidades, para activar, mantener y desactivar el enlace físico entre sistemas finales.
La capa física también es responsable por definir si la transmisión puede o no, ser realizada en los dos sentidos simultáneamente.

Modelo TCP/IP

Es el protocolo estandar para conexiones en redes corporativas. Las redes TCP/IP son ampliamente escalables, por lo que TCP/IP puede utilizarse tanto para redes pequeñas como grandes. Siendo un conjunto de protocolos encaminados, puede ser ejecutado en distintas plataformas, entre ellas Windows y Linux.

Tipos de redes según su topología

La topología de a red está definida por la forma en que se interconectan los diferentes equipos de trabajo con no o varios servidores de red.

Topología de bus

Está topología une a todos los equipos por un único medio de comunicación (bus), que reconoce todas las estaciones que forman lan red. Todas las estaciones se encuentran conectadas directamente a través de interfaces físicas llamadas tomas de conexión a un medio de transmisión lineal o bus.

Topología en anillo

En está topología todas las estaciones están conectadas formando un anillo. La red consta de na serie de repetidores.

Topología en estrella

En esta topología las estaciones están unidas a un nodo central, que realizan funciones de conmutación. Desde el nodo central sale el cableado para cada estación. Los equipos se comunican unos a otros a través del nodo central.

Topología de árbol

La topología en árbol es similar a la de bus, pero se permiten ramificaciones a partir de un punto llamado raíz, aunque no se permiten bucles. Este tipo de topología permite establecer una jerarquía clasificando las estaciones en grupos según el controlador al que estan conectadas.

Topología en malla

En esta topología todas las estaciones están conectadas entre sí. Su ventaja fundamental es que es muy fiable frente a falos, ya que si un nodo falla, sólo el queda aislado. Si una ruta entre dos nodos falla, siempre se puede encontrar un camino alternativo entre dos equipos.

Topología mixta

Cada nodo debe estar conectado, mínimo, por un enlace, pero no existen más retricciones. Esta topologíaes las más utilizada en redes que ocupan zonas geográficas amplias. Además permite la busqueda de rutas alternativas cuando falla algunos falla alguno de los enlaces.

Modelo Cliente-Servidor


Este modelo es uno de los principales usados en muchísimo servicios y protocolos de Internet, por lo que para todos aquellos que quieren aprender más sobre la Web y como funciona, entender el concepto del modelo Cliente-Servidor se vuelve algo indispensable.

Importancia del modelo Cliente-Servidor

La arquitectura del Cliente-Servidor tiene dos partes elementales diferentes, por un lado la parte del servidor y por otra parte del cliente o un grupo de clientes donde lo habitual es que un servidor sea una máquina bastante potente de datos, este se encarga de dar la respuesta demandada por el cliente.
Está arquitectura se inplica en diferentes modelos informáticos alrededor del mundo donde su propósito es mantener una comunicación de información entre diferentes entidades de una red mediante el uso de protocolos establecidos y el propio almacenaje. En está arquitectura el cliente suelen ser estaciones de trabajo que solicitan varios servicios al servidor, mientras quue un servidor es una máquina que actúa como depósito de datos y funcina como un sistema gestor de base de datos, este se encarga de dar la respuesta demandada por el cliente.
Su principal importancia es que permite conectar a varios clientes a los servicios que provee un servidor y como sabemos hoy en día, la mayoría de las aplicaciones y servicios tienen como gran necesidad que puedan ser consumidos por varios usuarios de forma simultanea.

Arquitectura de tres capas

Esta se utiliza para describir los sistemas de Cliente-Servidor en
donde el cliente solicita recursos y el servidor responde directamente
a la solicitud con sus propios recursos. Eso significa que el servidor no
requiere de una aplicación extra para propórcionar parte del servicio.

Arquitectura de dos capas

La capa del medio es denominada software intermedio, cuya tarea
es proporcinar los recursos solícitados pero que requiere de otro
para hacerlo. La última capa es el servidor de datos que proporciona
el servicio de aplicaciones, los datos necesarios para poder procesar
y generar el servicio que solicito el cliente en un principio.

Red

Es un conjunto de clientes, servidores y bases de datos unidosde una manera física o no física en el que existen protocolos de transmisión de información establecidos.

Cliente

Hace referencia a nu demandante de servicios, este cliente puede ser un ordenador, así como también una aplicación de información proveniente de la red para funcionar.

Servidor

Hace refencia a un proveedor de servicios, este servidor a se vez puede ser un ordenador o una aplicación informática la cual envía información a los demás agentes de la red.

Protocolo

Es un conjunto de normas, reglas y pasos establecidos de manera clara y concreta sobre el flujo de información, pueden ser mails, música, mensajes simples entre software, etc.

Base de datos

Son bancos de información ordenada, categorizada y clasificada que forma parte de la red, que son sitios de almacenaje para la utilización de servidores.

Servicios

Es un conjunto de información en busca las necesidades de un cliente donde esta información puede ser un mail, música, mensajes simples entre software, etc.
